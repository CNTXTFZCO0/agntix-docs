---
title: "Build your first agent"
description: "Create, configure, and test an Agentix AI agent from the dashboard in about five minutes."
---

An **agent** in Agentix is a configurable assistant — a model + a system prompt + an optional voice + optional tools and knowledge. This guide walks you through creating one in the dashboard, talking to it from the built-in test panel, and (optionally) attaching it to a knowledge store or tool.

## Prerequisites

- A signed-in dashboard account at [app.agntix.ai](https://app.agntix.ai). Sign up takes about 30 seconds; the default plan includes enough free quota to finish this guide.
- Admin role on your organization (Member accounts can view agents but not create them).

## Step 1: Open the Agents page

From the left navigation, click **Agents**. You'll land on `/agents`, which lists every agent in your organization. New organizations start empty.

<Frame caption="The Agents page. Click + Create Agent in the top right to start.">
  ![Empty Agents list](/images/guides/build-your-first-agent/01-agents-list-empty.png)
</Frame>

Click **+ Create Agent** to open the four-step creation wizard.

## Step 2: Pick a starting template

The wizard opens on **Step 1 — Templates**. You have two options:

- **Blank Template** — start from scratch (recommended if you have a specific use case in mind).
- **Templates carousel** — pre-built starting points like *Support Agent*, *Customer Service Agent*, *Banking Assistant*, *Appointment Setter*, or *Meeting Summarizer*. Each template pre-fills the system prompt and basic settings; you can edit everything afterwards.

<Frame caption="Step 1 of the Create Agent wizard.">
  ![Template picker](/images/guides/build-your-first-agent/02-create-step1-template.png)
</Frame>

For this guide, click the **Blank Template** card. The wizard advances to Step 2.

## Step 3: Fill in general settings

**Step 2 — General Settings** captures the basics:

1. **Voice Enabled** — leave off for a chat-only agent; turn on if you want the agent to make/receive phone calls. (You can flip this later — see [Voice calls](/guides/voice-calls).)
2. **Name** — at least 5 characters. Shown in lists, logs, and webhooks.
3. **Language** — primary language for the agent. Determines which TTS/STT models and voices are offered later.
4. **Role** — one-line summary of what the agent does (e.g. *Order-status assistant*).
5. **Description** — longer description; visible in the dashboard and analytics.

<Frame caption="General Settings — keep these short. The Role and Description help your teammates find this agent later; they aren't sent to the LLM.">
  ![General Settings form](/images/guides/build-your-first-agent/03-create-step2-general.png)
</Frame>

Click **Next**.

## Step 4: Write the system prompt

**Step 3 — System Prompt** is the most important step. The system prompt defines persona, tone, scope, and guardrails.

A good starter prompt for the order-status example:

```text
You help customers track their orders. Be concise and friendly.
- Ask for the order number if the user doesn't provide one.
- Never invent order status. If you don't have data, say so.
- If the user asks anything unrelated to orders, politely redirect.
```

<Frame caption="System prompt editor. Use the expand icon (top right of the textarea) to open the full-screen editor for longer prompts.">
  ![System Prompt step](/images/guides/build-your-first-agent/04-create-step3-prompt.png)
</Frame>

Click **Next**.

## Step 5: (Optional) Attach knowledge and tools

**Step 4 — Knowledge & Tools** lets you optionally attach:

- **Knowledge Base** — a previously-created knowledge store the agent will retrieve from. Skip for now if you don't have one; see [Chat with RAG](/guides/chat-with-rag) for how to create one.
- **Tools** — function/API tools the agent can invoke during a conversation. Skip for now; see [Tools](/guides/tools).

<Frame caption="Knowledge & Tools — both fields are optional. You can attach or detach at any time from the agent's detail page.">
  ![Knowledge and Tools step](/images/guides/build-your-first-agent/05-create-step4-attach.png)
</Frame>

Click **Create Agent**. You'll be redirected to `/agents/{id}` — the agent detail / edit page.

## Step 6: Test the agent

The detail page has two columns:

- **Left** — system prompt, model, and language. Edit any of these and click **Save Changes** in the top right.
- **Right** — tabs for **Calls**, **Chat**, **Functions**, **Multi-agents**, **Analytics**, **Widgets**. Each tab configures one slice of the agent.

In the header you'll see a green **Test Agent** button.

<Frame caption="Agent detail page. Click Test Agent (green button, top right) to start a live conversation.">
  ![Agent detail overview](/images/guides/build-your-first-agent/06-agent-detail-overview.png)
</Frame>

Click **Test Agent**. A LiveKit voice room or chat panel opens (depending on the **Voice Enabled** toggle). Type or speak a question your prompt should handle — e.g. *"What's the status of my order 1234?"* — and verify the agent stays in character.

<Frame caption="Close-up of the Test Agent button in the page header.">
  ![Test Agent button](/images/guides/build-your-first-agent/07-agent-test-call-button.png)
</Frame>

If you also want to share the agent without giving testers dashboard access, use the two small icons next to **Test Agent**:

- **Phone icon** — opens the SDK voice demo for this agent.
- **Speech-bubble icon** — opens the SDK chat demo for this agent.

Both URLs are shareable.

## Verify it works

Your agent is wired up if all of the following are true:

- It appears at `/agents` with the correct name and **Updated At** timestamp.
- Clicking **Test Agent** opens a working test panel without errors.
- The agent's replies follow the system prompt (correct persona, refuses out-of-scope questions).
- The **Analytics** tab on the detail page starts collecting session data after your first test.

## API alternative

If you'd rather create the agent over HTTP — for CI, scripts, or back-office tooling — every step above maps directly to the [Agents API](/api-reference/agents/create). The end-to-end curl flow is documented in the [Quickstart](/quickstart):

```bash
curl https://api.agntix.ai/v1/chat/agents \
  -H "x-api-key: $AGNTIX_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Order Status Bot",
    "modelId": "gpt-4o-mini",
    "systemPrompt": "You help customers track their orders. Be concise and friendly.",
    "config": { "temperature": 0.3, "maxOutputTokens": 200 }
  }'
```

See the full multi-step API flow (open a session, stream a message, attach a tool) in the [Quickstart](/quickstart) and the [Agents API reference](/api-reference/agents/create).
